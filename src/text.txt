## react intro

    What is React?

    1) React is a JavaScript library (not a framework) developed by Facebook (Meta) for building user interfaces (UIs), especially Single Page Applications (SPAs).

    2) It helps developers build reusable UI components that update efficiently when data changes.

    Instead of reloading a whole webpage when something changes, React updates only the parts of the UI that need to change.

    -------
    Why do we need React?

	It makes UI development faster, cleaner, and more efficient.

	Provides a better user experience with smooth updates without page reloads.

## useState Hook

    useState is a react hook that allows you to add state (data that can change over time)
    to a functional component.

    Normally ,in js function , variables disappear after the function runs.
    but with useState,react remembers values b/w  re-render.

    syntax:

        const[state,setstate]=useState(initialValue)

        state---        the current value of the state variable
        setstate---     A function that update thee state
        initialValue--- the starting value of the state variable

## Routing in react && nested routing

Routing means navigating between different pages in a web app without 
refreshing the browser.

In traditional website ,clicking a link reloads the whole page.

In react single  page Application , routing lets us changes the view without reloading.


        better prformance (no full reload)
        Smooth user experiance
        Control over navigation without server request

commend: npm install react-router-dom


structure of routing:

    <BrowserRouter>
    <Routes>
        <Route path="/address_path" elements={<Component/>}/>
        <Route path="/hamo" elements={<Home/>}/>
        <Route path="/about" elements={<About/>}/>
    </Routes>
    </BrowserRouter>


BrowserRouter:

    It is the main wrapper component for routing in reacct app.
    it enables navigation using the browser's URL .
    without this , routing will not work.

Routes:

    It contains all the <Route/> component.
    It tell react router with component to show for the which path
    Only one rout will match and render at a time.

Route:

    Definee a specific URL path and which component should load for the path.
